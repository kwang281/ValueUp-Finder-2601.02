# Q&A: Streamlit Cloud Key vs GitHub Actions Secret

## 1. 결론부터 말씀드리면
**"앱을 실행하는 것"**이 유일한 목적이라면, **GitHub Actions 설정은 필수가 아닙니다.**
Streamlit Cloud의 Secrets 설정만 제대로 되어 있으면 웹사이트는 정상적으로 작동합니다.

## 2. 왜 두 군데에 따로 설정하나요?
두 설정은 **사용되는 장소와 목적**이 전혀 다르기 때문입니다.

| 구분 | **Streamlit Cloud Secrets** | **GitHub Actions Secrets** |
| :--- | :--- | :--- |
| **용도** | **실제 앱 운영 (Runtime)** | **코드 검사 및 테스트 (CI)** |
| **언제 쓰이나?** | 웹사이트에 접속해서 버튼을 누를 때 | 코드를 GitHub에 업로드(Push) 할 때 |
| **목적** | 사용자가 기능을 이용하기 위함 | 개발자가 "이 코드가 고장나지 않았는지" 확인하기 위함 |
| **필수 여부** | **필수 (★★★★★)** | 선택 (권장) |

## 3. GitHub Actions가 필요한 이유 (선택 사항)
만약 GitHub Actions 설정을 하지 않으면, 코드를 수정해서 올릴 때 GitHub에서 **"테스트 실패(❌)"** 빨간불이 뜰 수 있습니다.
- 테스트 코드가 API Key를 불러와서 잘 작동하는지 확인하려고 시도하는데, 키가 없으니 에러가 나기 때문입니다.
- 하지만 이 테스트가 실패해도, Streamlit Cloud에 배포된 **실제 사이트는 아무 문제 없이 잘 돌아갑니다.**

## 4. 추천
- **"복잡한 것 싫고 사이트만 나오면 된다"** 👉 Streamlit Cloud 설정만 하시고, GitHub 쪽은 무시하셔도 됩니다.
- **"코드 관리도 체계적으로 하고 싶다"** 👉 두 곳 모두 설정해두시면, 나중에 코드를 고칠 때 실수를 미리 잡아줄 수 있어 좋습니다.
